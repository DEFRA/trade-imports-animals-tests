# journeys (E2E)

This folder is intended for journey tests.

## Notes

- A journey test (sometimes called a user journey test or end-to-end journey test) is a type of automated test that focuses on validating a complete flow through an application from the user’s perspective, rather than testing individual components or isolated features.
- This directory may start empty; Git won’t track empty folders, so the presence of this README is intentional until the first spec is added.
